Understanding How AI Systems Are Built

Before we can identify where bias emerges, we need to understand how machine learning systems are developed. Every ML system follows a seven-stage lifecycle, as detailed in the COMPAS development process:

Stage 1: Data Collection
What happens:
Teams gather data from the target population to train the algorithm. Decisions about what data to collect, from whom, and how to measure relevant outcomes.
Key decisions:
Which populations to include, what variables to measure, how to define the target outcome (e.g., 2-year recidivism vs 5-year).
Business impact:
These choices determine what the system can and cannot learn. Historical inequities in data collection become encoded into the algorithm.
Stage 2: Data Preparation
What happens:
Raw data is cleaned, structured, and prepared for training. Missing values are handled, outliers addressed, and data quality issues resolved.
Key decisions:
How to handle missing data, which data points to exclude, how to deal with inconsistencies across different data sources.
Business impact:
Preparation choices can systematically exclude certain groups or introduce measurement bias that affects fairness.
Stage 3: Feature Engineering
What happens:
Raw variables are transformed into features the algorithm can use. COMPAS used 137 variables including demographic info, criminal history, and questionnaire responses.
Key decisions:
Which variables to include as predictors, how to handle proxy variables (e.g., zip code as proxy for race), feature selection and weighting.
Business impact:
Feature choices can introduce indirect bias even when protected characteristics are explicitly excluded.
Stage 4: Model Training
What happens:
The algorithm learns patterns from training data. COMPAS likely used logistic regression and decision trees to identify relationships between predictors and recidivism.
Key decisions:
Algorithm selection, parameter tuning, optimization objectives (accuracy vs fairness), training/validation split ratios.
Business impact:
Training choices determine whether the model learns biased patterns from historical data and how it balances different performance metrics.
Stage 5: Model Validation
What happens:
The trained model's performance is assessed using test data. Northpointe focused on overall accuracy and predictive parity metrics.
Key decisions:
Which metrics to evaluate, what constitutes "good enough" performance, whether to test for bias across different demographic groups.
Business impact:
Limited evaluation can mask bias or unfairness issues that only become apparent after deployment.
Stage 6: Deployment
What happens:
The model is put into production use. COMPAS was deployed across multiple jurisdictions for pretrial, sentencing, and parole decisions.
Key decisions:
How judges should interpret scores, what use cases are appropriate, whether deployment matches original training scope.
Business impact:
Deployment scope expansion (using for bail when trained for sentencing) can introduce new bias sources and fairness concerns.
Stage 7: Monitoring
What happens:
Ongoing assessment of model performance in real-world use. ProPublica's analysis represented external monitoring that revealed bias issues.
Key decisions:
What metrics to track over time, how to detect bias drift, when to retrain or retire the model, transparency and accountability measures.
Business impact:
Inadequate monitoring allows bias problems to persist and compound, affecting fairness and organizational reputation.
