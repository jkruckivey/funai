Nine Types of ML Bias Framework

Bias can emerge at any stage of the ML development process. Understanding these categories helps identify potential issues before deployment.

Data-to-Algorithm Bias
  • Measurement Bias: Errors in how data is collected or measured
  • Representation Bias: Training data doesn't reflect target population
  • Aggregation Bias: Inappropriate grouping across different subpopulations
Algorithm-to-Users Bias
  • Learning Bias: Algorithm makes incorrect assumptions about patterns
  • User-Interaction Bias: How users interact with and interpret results
  • Evaluation Bias: Using wrong metrics or benchmarks for assessment
Users-to-Data Bias
  • Historical Bias: Past inequities encoded in training data
  • Deployment Bias: Misalignment between model design and actual use
  • Self-Selection Bias: Non-random participation in data generation
